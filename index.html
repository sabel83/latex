Build environment for Latex

Introduction

The goal was creating an easy to use (and re-use) tool for writing documentation and presentation in Latex. The target operating systems were GNU/Linux based systems, however it should work on all Unix based systems. Might work on Windows as well.

It makes heavy use of the GNU Make utility: the whole solution consists of a Makefile making it easy to deploy and use.

Requirements

The following software is required to use this solution:

  • GNU Make
  • GNU wget for updating the solution
  • Info-Zip for archiving Latex source
  • Latex
  • dvips
  • dvipdf
  • Command line ftp utility for uploading backups to a server
  • The basic Unix command line utilities

Usage

make compiles everything to every format
make [file format] compiles everything to specified format supported formats: dvi, ps, pdf
make clean_tmp deletes temporary files
make clean deletes temporary files and targets
make all rebuilds everything
make help display usage
make show compile everything and show them

program used for displaying can be overwritten:
  SHOW_CMD=<command>
preferred file format can be overwritten:
  SHOW_FORMAT=<format>
example:
  make show SHOW_CMD=kghostview SHOW_FORMAT=pdf
These values can be overwritten by defining environment variables as well, for example:
  export SHOW_CMD=kghostview
Default values:
  SHOW_CMD=acroread
  SHOW_FORMAT=pdf
make slides display a slideshow
Works similarly to "make show"
The environment variables it uses and their default values:
  SLIDES_CMD=xpdf -fullscreen -aa yes -aaVector yes
  SLIDES_FORMAT=pdf
make options display all options
make slides_template create a slides template latex source to start with the name of the generated template is the value of the TEMPLATE_NAME option.
update get the latest version of this makefile. Internet connection required.
zip cleans the code and creates a zip archive from it. The name of the zip file depends on the value of PROJECT_NAME.
project creates a projectName.txt file in which the value of the actual PROJECT_NAME is stored. It will be the default value for PROJECT_NAME
backup creates a zip archive and uploads it to a backup server defined by:
  FTP_HOST and FTP_USERNAME password has to be typed interactively.

Default values can be overridden by exporting the environment variables with the same name to the make process or by overriding them explicitly when invoking the make command:

make NAME=value
